# EPM Website: Launch Report

**Date:** 10 July 2026
**Result:** esapekkamattila.com serves the new site. The previous update to the old site was 2017.

## Objective and outcome

Replace a dormant Finnish-only WordPress installation (Suncomet, last updated 2017) with a modern bilingual personal site under full owner control, at zero hosting cost, editable without local tooling. Delivered: Astro static site on Cloudflare, EN at root and FI at /fi/, edited entirely through the GitHub web interface, deployed automatically on every commit.

## Stack and rationale

Astro 4.16 generates plain static files: nothing to patch, nothing to hack, no server runtime. This directly removes the failure mode of the old site, an unpatched WordPress accumulating security debt for nine years. GitHub (jalaton-lgtm/epm-website) is the single source of truth; the browser-upload workflow means the site is maintainable from any machine with no git CLI or Node installed. Cloudflare hosts it as a Workers-with-static-assets project at no cost and provides DNS, HTTPS, and analytics from the same account. Decap CMS is configured but not OAuth-wired; GitHub web editing covers current needs, so the OAuth work was deliberately deferred rather than blocking launch.

## Structural phase

Six items closed before any content work, on the principle that a real domain should never point at scaffolding.

A custom 404 page was built in the site's design language, bilingual on one page because Cloudflare serves a single root 404.html for all unmatched paths. This required a paired configuration change: wrangler.jsonc not_found_handling from "none" to "404-page", since the page file alone does nothing.

Open Graph and social meta tags went into BaseLayout.astro, one file propagating site-wide: canonical URLs, share titles and descriptions, a purpose-built 1200x630 share image cropped from the hero photo, locale tags per language, and Twitter card markup. Canonical URLs point at esapekkamattila.com by design, which prevents the pages.dev preview host from ever being indexed as the primary copy.

robots.txt references the sitemap (generated by @astrojs/sitemap) and disallows /admin/, the Decap CMS entry point. Analytics is Cloudflare Web Analytics: cookieless, no personal data stored, which is also why the privacy policy was skipped with a clear conscience. The site has no forms and no cookies; there is nothing to disclose. The analytics beacon is gated behind a token constant in BaseLayout, so an empty token ships nothing.

The three placeholder blog posts were set to draft: true rather than deleted. Deletion would have emptied the content directories, which GitHub then removes entirely; the drafts stay invisible, keep the directory structure, and serve as frontmatter reference for real posts.

## Content phase

The profile was rewritten from three third-person paragraphs to a first-person layered page: short lead and stats grid on top for skimmers, then narrative sections (Racing, Off the track, Now) and a named Roles list below. First person was chosen because it is a personal site, the writing samples are first-person, and third person reads as a press kit; the FI page and home previews switched with it so the voice has no seams.

Editorial discipline applied: the twenty-years motif cut from three uses to one, one self-labeling line removed, all em dashes eliminated, Tokyo 2021 corrected to the official Tokyo 2020. Every fact on the page was verified against independent public sources: the 2021 Bydgoszcz bronze, fifth at Kobe 2024, seventh at New Delhi 2025, three Paralympics, the NOC Athletes' Commission chair, the Olympic Committee board seat, and birth year against the IPC profile. The Roles list carries organisation names (Olympic Committee pairing highlighted first, SUL, Suomen Urheilijat ry, Abilis Foundation); the dormant Finnish Athletics Athletes role was omitted deliberately. The stats grid swapped Nationality for Born 1989, since FIN already appears under Based and the birth year otherwise existed nowhere on the page.

Known caveat, recorded honestly: the Finnish profile is a drafted translation published without a native review pass, by owner decision. It contains no factual errors but deserves a read at leisure.

## Cutover

Sequence: archive first, switch second. The complete Suncomet backup (files including wp-content/uploads, plus the SQL database) was confirmed downloaded before any DNS change, converting the old site from a liability into a restorable archive and removing the need for any waiting period before deletion.

The domain was added to Cloudflare, nameservers switched at Suncomet to daisy/walt.ns.cloudflare.com, and the mail and ftp records set to DNS-only before activation, since Cloudflare's proxy carries only web traffic. The custom-domain attachment required manual DNS records (root CNAME and www CNAME to epm-website.pages.dev, both proxied) because the automatic flow did not create them. www was added contrary to the original plan: the cost is thirty seconds and it catches habitual www typers.

Recorded as fact: any @esapekkamattila.com email address is non-functional as of cutover, since the MX target now resolves to the site proxy. No mailbox was confirmed to exist; if one turns out to matter, mail is recoverable with two DNS records.

## Verification discipline

Every code change was verified with a real local build before delivery, including a test build with all posts drafted to confirm the empty states render. Post-cutover, independent checks confirmed: NS delegation to Cloudflare, both hostnames resolving to proxy addresses, the live domain serving the new site with correct OG tags, the styled 404, old-URL redirects (date archives to home), www loading, and the Google site-verification TXT record answering publicly. Search Console ownership is verified and the sitemap submitted; first indexing data expected within a day.

## Incidents and platform behavior worth remembering

One GitHub-to-Cloudflare webhook silently missed a commit; the fix is any trivial commit to re-trigger a build. The deployment Retry button re-runs that entry's commit, not the latest, so the hash must be verified before using it. Local DNS caching showed the old site for hours after cutover while the world saw the new one; phone-on-mobile-data is the definitive test. Cloudflare automatically obfuscates mailto links via /cdn-cgi/l/email-protection; this is a feature. A visible photo placeholder shipped to the live home page and was caught by fetching the live HTML rather than trusting the checklist; live verification catches what checklists miss.
